


# תרגיל פרונטאנד: Movie Manager
בנו מסך React קטן שמתחבר לשרת הסרטים.
המסך צריך לאפשר טעינת כל הסרטים, סינון לפי ז׳אנר, חיפוש סרט לפי id, מחיקה, הוספה ועדכון.
השתמשו ב־`fetch`, ב־`useState`, ב־inputs פשוטים ובכפתורים.
אין צורך כרגע בעיצוב מורכב - הדגש הוא על החיבור בין הפרונט לבאק.


## מטרת המסך

לאפשר למשתמש:

* לראות את רשימת הסרטים
* לסנן סרטים לפי ז׳אנר
* לחפש סרט לפי `id`
* להוסיף סרט חדש
* לעדכן סרט קיים
* למחוק סרט

---

# מה צריך להיות במסך

## חלק 1 — כותרת

בחלק העליון של הדף הציגו כותרת:

```txt
Movie Manager
```

אפשר גם להוסיף תת־כותרת קצרה כמו:

```txt
Manage your movie library
```

---

## חלק 2 — כפתור לטעינת כל הסרטים

הוסיפו כפתור:

```txt
Load Movies
```

### מה הוא צריך לעשות?

בלחיצה עליו:

* תישלח בקשת `GET` ל־`/movies`
* רשימת הסרטים תחזור מהשרת
* הסרטים יוצגו על המסך




## חלק 3 — אזור להצגת הסרטים

הציגו את רשימת הסרטים שחזרה מהשרת.

לכל סרט אפשר להציג:

* `id`
* `title`
* `genre`
* `year`

### דוגמה לתצוגה

אפשר ככרטיסים או כרשימה פשוטה:

```txt
1 - Titanic - Drama - 1997
2 - Shrek - Animation - 2001
```

### מה מתרגלים?

* רינדור רשימה
* שימוש ב־`key`
* עבודה עם state של מערך



## חלק 4 בונוס — סינון לפי ז׳אנר עם query params

הוסיפו:

* שדה קלט לז׳אנר
* כפתור `Filter by Genre`

### מה צריך לקרות?

אם המשתמש כותב למשל:

```txt
Animation
```

בלחיצה על הכפתור:

* תישלח בקשת `GET` לכתובת:

```txt
/movies?genre=Animation
```

* יוצגו רק הסרטים מהז׳אנר הזה

### מה מתרגלים?

* controlled input
* בניית URL עם query params
* שימוש חוזר באותו state של `movies`

---

## חלק 5 — חיפוש סרט לפי id

הוסיפו:

* input עבור `movie id`
* כפתור `Find Movie`

### מה צריך לקרות?

אם המשתמש מזין `2`:

* תישלח בקשת `GET` ל־`/movies/2`
* על המסך תוצג תוצאה אחת

### אפשרות פשוטה

אפשר לבחור אחת מ־2 גישות:

* או להציג רק את הסרט שנמצא בתוך אזור הרשימה
* או להציג אותו באזור נפרד בשם `Selected Movie`


## חלק 6 — מחיקת סרט

ליד כל סרט ברשימה הוסיפו כפתור:

```txt
Delete
```

### מה צריך לקרות?

בלחיצה:

* תישלח בקשת `DELETE` ל־`/movies/:id`
* לאחר המחיקה, הרשימה על המסך תתעדכן

### מה מתרגלים?

* שליחת בקשת `DELETE`
* העברת `id`
* עדכון state אחרי מחיקה

### רמז:

אפשר:

* או לטעון מחדש את כל הרשימה מהשרת
* או לעדכן את ה־state מקומית בלי בקשה נוספת

---

## חלק 7 — טופס הוספת סרט חדש

הוסיפו טופס קטן עם שלושה inputs:

* title
* genre
* year

ומתחתיהם כפתור:

```txt
Add Movie
```

### מה צריך לקרות?

בלחיצה:

* תישלח בקשת `POST` ל־`/movies`
* המידע יישלח בתוך `body`
* אם ההוספה הצליחה, הסרט החדש יופיע ברשימה


## חלק 8 — טופס עדכון סרט

הוסיפו אזור נוסף לעדכון סרט קיים.

הטופס יכיל:

* input ל־`id`
* input ל־`title`
* input ל־`genre`
* input ל־`year`
* כפתור `Update Movie`

### מה צריך לקרות?

בלחיצה:

* תישלח בקשת `PUT` ל־`/movies/:id`
* הנתונים החדשים יישלחו ב־body
* הסרט המעודכן יוצג במסך / ברשימה

### מה מתרגלים?

* `PUT`
* route param + body ביחד
* עבודה עם מספר שדות בטופס

---

## חלק  9 בונוס — הצגת הודעות למשתמש

הוסיפו אזור להודעות.

הציגו שם:

* הודעת הצלחה
* הודעת שגיאה

### דוגמאות

```txt
Movie deleted successfully
Movie added successfully
Movie not found
Unauthorized
```

### מה מתרגלים?

* state של `message`
* הבחנה בין הצלחה לשגיאה
* `try/catch` בצד הפרונט

---

## חלק 10 — מצב טעינה

כאשר נשלחת בקשה לשרת, הציגו טקסט:

```txt
Loading...
```

וכשהבקשה מסתיימת — הסתירו אותו.

### מה מתרגלים?

* state של `loading`
* UX בסיסי

---

# דרישות טכניות לתרגיל

## להשתמש ב־React בסיסי

התרגיל צריך להיכתב בגישה פשוטה עם:

* `useState`
* event handlers
* `fetch`

לא חייבים:
* context
* ספריות UI

---

## לעבוד עם fetch

כל הבקשות יישלחו עם `fetch`.

### סוגי הבקשות שיהיו:

* `GET`
* `POST`
* `PUT`
* `DELETE`

---

## לעבוד עם JSON

ב־`POST` ו־`PUT` יש לשלוח:

* header של `Content-Type: application/json`
* body בפורמט JSON
